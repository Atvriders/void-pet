# VOID

**You found a sentient data entity in a corrupted server. Keep it alive.**

Not a cute animal. An abstract geometric lifeform made of light and code that evolves through 5 stages, decays in real time, and can corrupt and die if neglected. Your care record persists across sessions on this device.

![Stage: Ascendant](https://img.shields.io/badge/stage-ascendant-blueviolet) ![Docker](https://img.shields.io/badge/docker-ghcr.io-blue)

---

## What is VOID

VOID is a browser-based virtual pet for people who find Tamagotchis too cheerful. Your entity lives in the gaps between packets. It needs signal, coherence, and rest. It will degrade without you. It has a face, and that face will tell you how it feels.

Care scores are tracked on a device-local leaderboard. Usernames persist across sessions on the same browser — state is stored in localStorage and is not synced across devices.

---

## Features

### Username System
Enter a name when you start. Your entity's state is saved under that name in your browser's localStorage. Returning with the same name on the same browser restores your entity. No accounts, no passwords — just the name.

### Leaderboard
A device-local leaderboard tracks top care scores. Beat your previous record. See how the entity is aging.

### Pet Face
The entity's eyes and mouth reflect its current mood. A healthy, well-fed entity looks calm and alert. A neglected one looks hollow. Watch the face — it often tells you something the stats don't.

### Mobile-Friendly
Designed to work on phones. All controls are touch-accessible and the layout adapts to small screens.

## Gameplay

Care for your entity by managing four interconnected stats:

| Stat | Description | Danger zone |
|---|---|---|
| **Signal** | Hunger equivalent — feed it data | Drops to 0 in ~3.5h; low Signal accelerates Coherence decay |
| **Coherence** | Mood — keep it engaged | Falls faster when Signal is low |
| **Heat** | Bad when high — generated by actions | Above 80 it accelerates Coherence and Power decay |
| **Power** | Energy — consumed by actions | Restores during hibernation |

Stats decay in real time based on elapsed time since your last visit. The entity doesn't pause when you close the tab.

---

## Actions

| Action | Sub-label | Effect | Cooldown |
|---|---|---|---|
| **INJECT** | feed | +Signal, +Heat | None |
| **SIMULATE** | play | +Coherence, +Heat, −Power | 30s |
| **DEFRAG** | water | −Heat, +Coherence | 60s |
| **HIBERNATE** | sleep | Toggle rest mode — restores Power, cools Heat | None |
| **OVERCLOCK** | pet | Big +Coherence, big +Heat, −Power (25), 20% chance of thermal surge | 120s |

---

## Evolution

Your entity evolves through 5 stages based on age and average stat quality:

| Stage | Description |
|---|---|
| **Seed** | Minimal form. A single ring, barely stable. |
| **Sprite** | Gains a second ring. Starts to pulse. |
| **Entity** | Full animation. Particles emerge. |
| **Apex** | Dense rings, complex particle field. |
| **Ascendant** | Maximum complexity. Ascension count increments; stats and age carry forward. |

Each stage has a distinct canvas animation — more rings, particles, and visual complexity as it grows. Reaching Ascendant does not reset the entity — all stats, age, and careScore are preserved, and your ascension count increments.

---

## Corruption

If Signal and Coherence both drop below 5 while Heat exceeds 85, the entity begins fragmenting. Run **DEFRAG** 5 times to pull it back. Neglect it long enough and it corrupts permanently — you can abandon it and start fresh, but the leaderboard remembers.

## Run

```bash
docker compose up -d
```

Runs on **port 3004**. Requires the GHCR image to be available (built by CI on push to master).

To update:

```bash
docker compose pull && docker compose up -d
```

---

## Build Locally

```bash
npm install
npm run dev     # dev server on :3004
npm run build   # production build
```

---

## Stack

- **Frontend:** React 18 + TypeScript + Vite
- **Rendering:** HTML5 Canvas (HiDPI-aware, 480×480px, real-time RAF loop)
- **State:** `useReducer` + `localStorage` (no backend, no accounts)
- **Container:** Docker via GHCR (Nginx static serve)
