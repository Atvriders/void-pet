# VOID

**You found a sentient data entity in a corrupted server. Keep it alive.**

VOID is a browser-based virtual pet for people who find Tamagotchis too cheerful. Your entity is an abstract geometric lifeform made of light and code — not a cute animal. It evolves through 5 stages, decays in real time, and can corrupt and die if neglected. Progress is saved server-side and shared across devices via a username system. A global leaderboard tracks top care scores.

---

## Quickstart

```bash
docker compose up -d
```

Open **http://localhost:3004**. Enter a username to begin.

To update to the latest build:

```bash
docker compose pull && docker compose up -d
```

## Architecture

Two services run via Docker Compose:

| Service | Image | Port | Role |
|---|---|---|---|
| `void-pet` | `ghcr.io/atvriders/void-pet:latest` | 3004 (host) | Nginx serving the React SPA. Proxies `/api/` requests to the API service. |
| `api` | `ghcr.io/atvriders/void-pet-api:latest` | 3005 (internal) | Express + SQLite API. Stores pet states and leaderboard scores in a named Docker volume. |

The API service has a healthcheck on `/api/health`. The frontend container waits until the API is healthy before starting (`condition: service_healthy`).

Data is persisted in a named Docker volume (`void_data`) mounted at `/data` inside the API container.

## Username System

On first load you are prompted to enter a username (2–16 characters, letters/numbers/underscores). This username:

- Identifies your save slot on the server
- Is stored in `localStorage` so you don't have to re-enter it on the same device
- Can be reused on any device or browser to resume the same entity

There are no accounts or passwords. Anyone who knows a username can load that entity. Choose something unique.

## Stats

Your entity has four stats, each ranging from **0 to 100**. They decay continuously in real time while the tab is open. Stats do **not** decay while the tab is closed — this is an active-play design.

| Stat | Icon | Description | Danger zone |
|---|---|---|---|
| **Signal** | ◈ | Hunger equivalent — feed data packets to keep it high | Below 20 accelerates Coherence decay |
| **Coherence** | ❋ | Mood and stability | Below 30 triggers `sad` mood; decays faster when Signal is low or Heat is high |
| **Heat** | ▲ | Thermal load — generated by actions, bad when high | Above 80 accelerates Coherence and Power decay; triggers `overheating` mood |
| **Power** | ⚡ | Energy — consumed by actions, restored during hibernation | Below 15 triggers `tired` mood; gates SIMULATE and OVERCLOCK |

**Care score** accumulates over time based on stat health. Each healthy stat adds 1 point per minute:

| Condition | Contribution |
|---|---|
| Signal > 60 | +1 pt/min |
| Coherence > 60 | +1 pt/min |
| Heat < 40 | +1 pt/min |
| Power > 40 | +1 pt/min |

Maximum rate is 4 points per minute. Care score never decreases.

## Actions

Each button shows colored stat effect glyphs at the bottom — green means the stat improves, orange means it takes a hit.

| Button | Sub-label | Effects | Cooldown | Requirements |
|---|---|---|---|---|
| **INJECT** | feed | ◈+ ▲+ | None | Not sleeping, not corrupted |
| **SIMULATE** | play | ❋+ ▲+ ⚡− | 30 s | Not sleeping, not corrupted, Power ≥ 15 |
| **DEFRAG** | water | ▲− ❋+ | 60 s | Not sleeping (available while corrupted for recovery) |
| **HIBERNATE** | sleep / wake | ▲− ⚡+ (passive, while sleeping) | None | Not corrupted |
| **OVERCLOCK** | pet | ❋+ ▲+ ⚡− | 120 s | Not sleeping, not corrupted, Power ≥ 25 |

When a button is pressed, the creature reacts with a canvas burst animation and floating stat-change labels rise above it:

| Action | Burst effect |
|---|---|
| INJECT | 6 ◈ glyphs converge inward from the edges |
| SIMULATE | 8 L-shaped circuit traces expand outward with node dots |
| DEFRAG | 3 cyan cooling ripple rings + 8 outward particles |
| HIBERNATE | 8 stars orbit the creature and fade |
| OVERCLOCK | 5 jagged lightning bolts fire outward + white flash |

**OVERCLOCK** has a 20% chance of a thermal surge: +60 Heat instead of +35.

During **hibernation**: all other actions are disabled, stat decay slows, Heat cools faster, and Power regenerates passively.

## Moods

The entity's face and the status label reflect its current condition. Evaluated top-down — first match wins.

| Mood | Label | Trigger |
|---|---|---|
| corrupted | ⚠ CORRUPTION DETECTED | Stage is `corrupted` |
| sleeping | HIBERNATING | Hibernation is active |
| overheating | THERMAL OVERLOAD | Heat > 80 |
| hungry | SIGNAL LOST | Signal < 20 |
| tired | LOW POWER | Power < 15 |
| happy | STABLE ◈ HARMONISED | Coherence > 70, Signal > 60, Heat < 40 |
| sad | COHERENCE DEGRADING | Coherence < 30 |
| neutral | NOMINAL | *(none of the above)* |

The entity's canvas animation, face expression, and UI accent colour all change with mood.

The creature canvas is always animated with several layers:

- **Matrix rain** — columns of falling katakana and symbols cascade behind the entity, tinted to the current mood hue
- **Scanline sweep** — a soft green gradient band drifts continuously down the canvas
- **Blinking** — the entity blinks at random intervals (every 2.5–6.5 seconds), with a smooth 150 ms close/open
- **Stage-specific effects** — Apex stage adds occasional lightning arcs; Ascendant cycles through all hues

## Evolution

The entity evolves through 5 stages. Stage is evaluated every second during the game loop. Evolution requires the entity to be awake and not corrupted.

Stat average formula: `(Signal + Coherence + (100 − Heat) + Power) / 4`

| Stage | Min Age | Min Avg Stat | Min Care Score |
|---|---|---|---|
| **Seed** | *(start)* | — | — |
| **Sprite** | 15 min | 50 | 0 |
| **Entity** | 60 min | 55 | 50 |
| **Apex** | 180 min | 60 | 200 |
| **Ascendant** | 480 min | 65 | 500 |

Reaching **Ascendant** increments your ascension counter. Stats, age, and care score carry forward — there is no penalty reset. The entity stays at Ascendant until the tab is closed or it corrupts.

Each stage has a distinct canvas rendering: more rings, orbital particles, and visual complexity as the entity advances.

---

## Corruption and Recovery

If **Signal < 5 and Coherence < 5 and Heat > 85** simultaneously for more than **20 consecutive seconds**, the entity becomes corrupted.

While corrupted:
- The game loop pauses (no further decay)
- Most actions are disabled
- A recovery counter shows: ⚠ CORRUPTION — run DEFRAG ×5 to recover

**To recover:** run DEFRAG 5 times. Each use reduces the counter by 1. On completion, the entity resets to Seed stage with partial stats: Signal 40, Coherence 40, Heat 50, Power 40.

If you don't want to recover, use the **ABANDON** button in the top bar to delete the entity and start fresh. The leaderboard retains its peak score entry.

---

## Leaderboard

The global leaderboard tracks the top 50 care scores across all usernames. Scores are submitted to the server automatically as care score increases. Only a new personal best replaces your existing entry.

Open the leaderboard via **⊞ BOARD** in the top bar. It shows username, care score, current stage, entity age, and ascension count.

A green dot appears next to any username whose pet was saved within the last 60 seconds — indicating they are currently active. The leaderboard query joins the `pets` table on `updated` timestamp to determine this without a separate heartbeat mechanism.

---

## Renaming Your Entity

The entity's display name (default `ENTITY_01`) can be changed at any time with the **✎ RENAME** button in the top bar. This is separate from your username — the display name is cosmetic and stored in your save state.

---

## Saving

Pet state is saved to the server automatically on every state change (approximately once per second while the tab is active). On load, the server state is fetched and applied before the game loop starts, so returning users never see a zeroed-out entity.

Username is saved in `localStorage` so the entry screen is skipped on return visits.

## API Endpoints

The API runs on port 3005 internally and is proxied via Nginx under `/api/`.

| Method | Path | Description |
|---|---|---|
| `GET` | `/api/health` | Health check. Returns `{ ok: true }`. |
| `GET` | `/api/pet/:username` | Load saved pet state. 404 if not found. |
| `PUT` | `/api/pet/:username` | Save pet state (full JSON body). |
| `DELETE` | `/api/pet/:username` | Delete a pet save. |
| `GET` | `/api/leaderboard` | Top 50 scores, ordered by care score descending. |
| `POST` | `/api/scores` | Submit a score. Only stored if higher than existing entry for that username. |
| `DELETE` | `/api/scores/:username` | Remove a leaderboard entry. |

## Local Development

```bash
# Frontend (Vite dev server on :5173, proxies /api/ to :3005)

npm run dev

# API (in a separate terminal)
cd server
npm install
node index.js   # listens on :3005
```

Production build:

```bash
npm run build   # outputs to dist/
```

## Stack

| Layer | Technology |
|---|---|
| Frontend | React 18 + TypeScript, bundled with Vite |
| Game loop | `useReducer` state machine, RAF-driven tick at ~1 Hz |
| Rendering | HTML5 Canvas (HiDPI-aware, 480×480 logical px) |
| Persistence | Express 4 + better-sqlite3, SQLite database on a Docker volume |
| Serving | Nginx (static SPA + `/api/` reverse proxy) |
| Container | Docker Compose, images published to GHCR |
